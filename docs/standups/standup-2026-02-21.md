### Implicit Map Syntax for Prologos

Implement implicit map syntax for Prologos using significant whitespace. This allows writing map literals in an indentation-based style similar to YAML, while remaining homoiconic by parsing to the same AST as explicit `{:key value}` maps.

#### Syntax Rules

1. **Nested map**: Line with keyword only, followed by indented block
   ```
   :address
     :city "NYC"
     :zip 10001
   ```
   Parses as `{:address {:city "NYC" :zip 10001}}`

3. **Vector values**: Use `@[...]` syntax (unchanged from explicit maps)
   ```
   :tags @[:admin :active]
   ```

4. **Computed values**: Use `[...]` command syntax (unchanged)
   ```
   :created-at [now]
   :full-name [concat first-name " " last-name]
   ```

5. **List of items**: Use `-` prefix for vector elements
   ```
   :users
     -
       :name "Alice"
     -
       :name "Bob"
   ```
   Parses as `@[:users @[{:name "Alice"} {:name "Bob"}]]`

#### Full Example

```prologos
def app-config
  :name "MyApp"
  :date [now]   ;; date is computed dynamically
  :version "1.0.0"
  :server
    :host "localhost"
    :port 8080
    :ssl
      :enabled true
      :cert-path "/etc/ssl/cert.pem"
  :database
    :url [env "DATABASE_URL"]
    :pool-size 10
  :features @[:auth :logging :caching]
  :admins
    -
      :name "Alice"
      :role :super
    -
      :name "Bob"
      :role :regular
```

Parses to equivalent:

```prologos
def app-config
  {:name "MyApp"
   :date [now]
   :version "1.0.0"
   :server {:host "localhost"
            :port 8080
            :ssl {:enabled true
                  :cert-path "/etc/ssl/cert.pem"}}
   :database {:url [env "DATABASE_URL"]
              :pool-size 10}
   :features @[:auth :logging :caching]
   :admins @[{:name "Alice" :role :super}
             {:name "Bob" :role :regular}]}
```
