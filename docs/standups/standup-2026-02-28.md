## Standard Library Design — Deep Research + Literate Book System

Completed Phase 1 deep research: surveyed 10 language ecosystems (Go, Java, Rust, Haskell, Clojure, Elixir, Lean 4, Idris 2, SWI-Prolog, Mercury). Identified 8 problems with current library organization and 8 design principles.

Through iterative design, evolved from directory-based reorganization to a **Literate Book System** — separating the dependency DAG from human presentation order. Key insight: `.prologos` itself IS the documentation format. Comments are prose, `module` directives mark compilation unit boundaries.

Built Phase 1 proof of concept:

-   `lib/prologos/book/OUTLINE` — reading order manifest (named chapters, not numbered)
-   `lib/prologos/book/equality.prologos` — first chapter (7 modules, with prose)
-   `tools/tangle-stdlib.rkt` — text-processing tangler (~175 LOC)
-   Verified: tangled output has identical code to originals, 471+ tests pass

(commit ce12ac7)
