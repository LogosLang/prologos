Key implementation decisions and fixes:
- *Parametric impl dispatch fix* (=macros.rkt=): =impl HasTop (FlatVal A)= was routed
  to monomorphic handler because it has no =where= clause but has a compound type arg.
  Added =has-compound-type-args?= check to dispatch condition.
- *QTT multiplicity fix*: Inline lambdas referencing prelude functions (e.g. =or=) in
  =net-new-cell= merge position triggered multiplicity violations. Workaround: define
  merge as separate =def= with explicit type annotation (matching integration test pattern).
- *HasTop trait + BoundedLattice bundle*: =HasTop= provides =top= for contradiction
  detection. =BoundedLattice= is a bundle of =Lattice + HasTop=. Instances for
  Bool, FlatVal, Interval.
- *Deferred: =new-lattice-cell= generic wrapper*: Resolving =Lattice-bot=/=Lattice-join=
  implicit type params inside generic closures hits a meta-resolution limitation.
  Users call =net-new-cell= directly with explicit bot/merge values.
