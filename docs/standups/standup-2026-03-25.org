Yes — that's a cleaner separation. PRN is the THEORY (hypergraph/hyperlattice rewriting as a unifying formalism). PPN, SRE, PReductions are APPLICATIONS that each contribute instances and insights back to the theory.

The relationship:

```markdown
PRN (theory — hyperlattice rewriting formalism)
  "What ARE the universal primitives for typed rewriting on propagator networks?"
  Informed by: all application tracks
  Delivers: foundational research, categorical grounding, shared abstractions

PPN (application — grammar/parsing)
  "How do we express parsing as lattice rewriting?"
  Contributes to PRN: parse lattice design, ambiguity resolution patterns,
    bidirectional information flow (type-directed disambiguation)
  Consumes from PRN: rewrite rule registration, fixpoint semantics

SRE (application — structural reasoning)
  "How do we express structural decomposition/composition as rewriting?"
  Contributes to PRN: form registry = rewrite rules, relation-parameterized
    dispatch, the ctor-desc model
  Consumes from PRN: when PRN provides better abstractions than what SRE
    built ad-hoc

PReductions (application — reduction/optimization)
  "How do we express β/δ/ι reduction and optimization as rewriting?"
  Contributes to PRN: e-graph saturation patterns, tropical cost extraction,
    confluence analysis
  Consumes from PRN: rewrite rule infrastructure, termination guarantees

BSP-LE (application — logic engines)
  "How do we express logical inference as lattice rewriting?"
  Contributes to PRN: bilattice rewriting, NAF as non-monotone rewrite,
    tabling as memoized rewriting
  Consumes from PRN: stratified rewriting, ATMS-guided search
```

PRN isn't a track you "implement" first. It's a research series that CRYSTALLIZES as the applications teach us what the universals are. Just like NTT crystallized from PM Tracks 1-8. You build the applications, observe the patterns, and PRN is where you NAME and FORMALIZE those patterns.

The practical consequence: PRN doesn't block anything. PPN Track 0 proceeds with its own lattice design, informed by SRE's existing form registry. SRE Track 2C proceeds with elaborator changes. PReductions Track 0 proceeds with e-graph research. When two applications discover they need the same primitive (rewrite rule registration, fixpoint termination, cost-weighted selection), that's a PRN research item — formalize the shared primitive, then both applications adopt it.

The interleaving rhythm:
- PPN builds something → discovers a pattern → PRN research note captures it
- SRE builds something → discovers the SAME pattern → PRN formalizes it as shared
- PReductions needs it → consumes the PRN formalization instead of reinventing

This is exactly how NTT worked: Track 8 built bridges → NTT named "Galois connection." SRE built decomposition → NTT named "polynomial functor dispatch." The naming happened AFTER the building, and the named abstractions then made future building faster.
