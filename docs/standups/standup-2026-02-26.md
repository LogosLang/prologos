## Relational Language ‚Äî Surface Syntax Design

Iterative design session covering the full relational language surface syntax:

-   **Solver design**: `solve=/=explain` split. `solve` returns `Seq (Map Keyword Value)` (bare bindings, zero provenance overhead). `explain` returns `Seq (Answer Value)` with bundled bindings + derivation trees. Both have `-with` variants accepting `{...}` map merge overrides.
-   **`solver` top-level form**: Configuration entity carrying execution, strategy, tabling, provenance, timeout. `default-solver` is module-level, shadowable. Completes the `spec=/=session=/=solver` triple.
-   **Facts syntax**: `||` for disjunctive fact blocks within `defr`, `&>` for rules. Facts are bodiless clauses inside `defr`, not a separate top-level form.
-   **Schema design**: `schema` as named closed-map type (like Rust structs / Clojure spec). Completes the `spec=/=schema=/=session` specification triple. Schema-annotated `defr` gets positional facts with type checking.
-   **Logic variable conventions**: Bare names in rule bodies (no `?` prefix), mode contracts (`+in=/`-out=) only in argument signatures. Correct Mercury conventions (`+` = input, `-` = output).
-   **Multi-arity `defr`**: `|` clause separator (matching `defn` multi-body pattern).
-   **Relational ‚Üí database isomorphism**: Facts ‚Üî rows, predicates ‚Üî tables, rules ‚Üî composable views, tabling ‚Üî materialized views, propagators ‚Üî incremental view maintenance, ATMS ‚Üî provenance.

<a id="orge59204d"></a>

### Key Decisions

-   **Shadow mirroring, not full switchover**: The propagator network mirrors the meta-store via hooks. Meta-store remains primary for correctness; network validates and provides precise fork/restore for speculation. Full switchover (70+ call sites, 14 files) deferred.
-   **`with-speculative-rollback` as single abstraction**: One helper wraps all 4 speculation sites (3 in `typing-core.rkt`, 1 in `qtt.rkt`). Saves meta-state + forks network, runs thunk, restores both on failure.
-   **Error enrichment at error layer**: E1006 union exhaustion errors produced by `check/err` in `typing-errors.rkt` ‚Äî zero changes to `typing-core.rkt` or `qtt.rkt`. `whnf` on expected type, `flatten-union-local` to decompose nested unions.
-   **Speculation failure recording**: `current-speculation-failures` collects branch failure labels per command. Infrastructure ready for future consumption.

### Key Fixes

-   `fresh-meta` returns `(expr-meta id)`, not raw symbol ‚Äî must extract with `(expr-meta-id m)` before `solve-meta!`

-   `process-def-group`: re-init shadow + speculation tracking per clause to prevent teardown across multi-body defs
-   `maybe-shadow-validate!`: defensive check on `net-box` before calling `shadow-validate!`
