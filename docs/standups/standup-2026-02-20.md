# Standup — 2026-02-20

## Yesterday (Feb 19–20)

### Test Infrastructure Overhaul

The test suite had grown to ~2700 tests across 80+ files with no visibility into per-file timing, no regression detection, and one pathological test (`test-pipe-compose.rkt`) that ran for 60+ minutes due to a numeric literal bug in a fixture. We fixed that and then built proper infrastructure:

1. **Targeted test runner** (`tools/run-affected-tests.rkt`) — reverse-dependency DAG so `racket tools/run-affected-tests.rkt` only runs tests affected by changed files. Full suite via `--all`.
2. **Benchmarking infrastructure** (`tools/benchmark-tests.rkt`) — per-file timing to JSONL, trend reporting, auto-regression detection, `--slowest N` for finding whale files.
3. **DearPyGui dashboard** — interactive visualization of test timings over runs, with crosshair annotations and error log view.
4. **Test skip mechanism** (`.skip-tests`) — config file to bypass known-broken tests without code changes.
5. **Bytecode pre-compilation** — `raco make driver.rkt` before runs cut per-subprocess preamble from ~22s to ~1s. Full suite dropped from ~400s to ~104s.
6. **Whale file splits** — 4 large test files split into 11 smaller ones for better parallelism.

**Net result**: Suite went from ~400s wall time with a 60-min blocker → **~104s**, 90 files, 2717 tests, all passing.

### Prelude System with Auto Imports

Implemented a Haskell/Clojure-style prelude so `ns foo` auto-imports everything needed for typical programs — Nat, Bool, List, Option, Result, Pair, core combinators, all traits (Eq, Ord, Add, Sub, Mul, Num, etc.) and their instances. Library modules use `:no-prelude` to avoid circularity. Defined as a Racket-side requires list in `namespace.rkt` to sidestep the `:refer-all` FQN mismatch problem.

## Today (Feb 20)
### Core Data Structures — Transient Builders (Phase 2d)

Added Clojure-style transient builders for O(n) batch construction of persistent collections. Previously, building a PVec/Map/Set element-by-element was O(n log n) due to copy-on-write at each step. Now: `transient` → mutate → `persist!` for O(1) amortized per-op. 20 new AST nodes, `with-transient` macro, 38 tests.
