#### Phase 7b: Broadcast `.*field` Syntax (`0a61cba`)

Full 13-file pipeline for `expr-broadcast-get` AST node:
- **Reader**: `.*ident` tokenized as `broadcast-access` token type
- **Macros**: `$broadcast-access` sentinel desugared to `(broadcast-get target :field...)` with chained dot-access support for deep broadcast (`.*a.b` → `broadcast-get` with `:a :b`)
- **Parser**: `broadcast-get` keyword handler → `surf-broadcast-get` surface struct
- **Surface-syntax**: `surf-broadcast-get` struct (target, fields, srcloc)
- **Syntax**: `expr-broadcast-get` struct (target, fields)
- **Elaborator**: `surf→expr` conversion, symbol fields to `expr-keyword`
- **Typing-core/QTT**: fresh meta result type, target usage tracking
- **Reduction**: walks cons/nil list structure (handles bare, qualified, and typed constructors), applies chained `map-get` for each field keyword
- **Substitution/Zonk/Pretty-print**: structural traversal over target

**Key design decision**: Pivoted from map+lambda desugaring to dedicated AST node because `list::map` with untyped lambda `[fn [x] [map-get x :field]]` fails inference — `map-get` doesn't constrain `x`'s type. The `expr-broadcast-get` node avoids this inference gap entirely.

**Acceptance file**: 31 results (up from 26), 0 errors. New tests: `scores.*x` → `'[1 2 3]`, `scores.*y` → `'[10 20 30]`, `nested-maps.*a.b` → `'[100 200 300]`.

#### Phase 7c: Key Renaming `^` Syntax (`c7bb09f`)

Key renaming in path segments — SQL `SELECT ... AS` equivalent:
- **Reader**: `^` added to `ident-continue?` so `name^alias` tokenizes as single identifier
- **Parser**: `validate-selection-paths` splits segments on `^` into rename pairs `(cons #:original #:alias)`
- **Elaborator**: get-in branching paths use renamed key for result map entry (`seg->result-kw`), original key for navigation (`seg->nav-kw`); update-in rejects rename pairs with parse error

**Example**: `[get-in m :x^a :y^b]` → `{:a 1, :b 2}` — extracts `:x` and `:y` but uses `:a` and `:b` as result keys.

**Deferred**: Dot-access brace expansion with `^` (`m.{name^alias}`) needs reader-level brace-expansion awareness; broadcast+rename composition.

**Acceptance file**: 33 results (up from 31), 0 errors.

#### QTT Fix: expr-union in inferQ (`08fa7d1`)

- **Root cause**: `inferQ` in qtt.rkt had no `expr-union` case. When heterogeneous maps undergo type widening (e.g., `String` + `Int` → `<String | Int>`), union types appear as embedded type args in `expr-map-empty`. QTT hit the default error case → "Multiplicity violation".
- **Fix**: Added `expr-union` case following the pattern of other type formers (Pi, Sigma, Eq, Map, Set) — infer both branches, combine usages, return `(tu (expr-Type (lzero)) ...)`. Single 7-line addition.
- **Result**: User's `app-config` map (nested maps, dynamic function calls, PVec, lists) now elaborates successfully.
- **Remaining §U issues** (pre-existing, not path-related):
