### Discussion: CRDT Hypothesis for Endo Formula Retention

Analyzed an Endo maintainer's hypothesis: "the formula system is missing a CRDT for
managing retention of local formulas on a pair of connected peers that must be able to
progress when partitioned, notably for deletion of references, as this is the basis
for revocation."

**Analysis**:
- Validated: CTP_DROP reference counting is a distributed consensus protocol —
  fails under partition (lost DROP = leaked formula, premature DROP = dangling reference)
- The partition problem is fundamental: reference counting requires coordination,
  but coordination requires connectivity
- CRDTs (join-semilattice based) are the right lens: convergent state without coordination
- Revoke-wins semantics aligns with OCap principles (revocation is a security primitive)

**CRDT design proposed**:
- `RetentionState` lattice: `{alive, revoked}` with `join(alive, revoked) = revoked`
- Epoch-based hybrid: `(max-epoch, revoke-wins-within-epoch)` for partition recovery
- Session type protocol for CRDT retention sync

### Discussion: Model Checking ↔ Propagator Networks

Extended brainstorming session exploring the structural isomorphism between:
- Prologos's propagator-based session type checking + effect ordering lattice + ATMS
- CTL model checking + modal μ-calculus + fixpoint computation

**Key correspondences identified**:

| Propagator Concept | Model Checking Concept |
|---|---|
| `run-to-quiescence` | Least fixpoint (μ) iteration |
| Propagator cell | State label / atomic proposition |
| ATMS worldview | Execution path |
| All worldviews agree | Universal path quantifier (A) |
| Some worldview satisfies | Existential path quantifier (E) |
| `check-session-completeness` | AF end (inevitable termination) |
| Contradiction in cell | Safety violation (¬AG φ) |
| Galois connection (α, γ) | Abstract model checking |
| Effect ordering lattice | Temporal ordering of states |

**Tier-level accessibility model**:
- Tier 4+: Expert tools (SPIN, TLA+, NuSMV) — model checking today
- Tier 3: Property annotations (user writes `property always ...`)
- Tier 2: Inferred from types (session types → safety properties automatically)
- Tier 1: Invisible/automatic (the compiler IS the model checker)

**Key insight**: Prologos is already at Tier 2 for safety properties. The gap to Tier 2
for liveness is the greatest fixpoint (ν) — which propagators naturally support via
initialized-to-top cells with monotone-decreasing updates.
