### Decisions & Lessons Learned

1. **Session-name-as-implicit-label**: Anonymous `(Mu body)` inside `(session Loop ...)` uses `'Loop` as the recursion label. This makes the common pattern `(session Loop (Mu (Send Nat (SVar Loop))))` work naturally — no need for `(Mu Loop (Send Nat (SVar Loop)))` explicit label syntax.

2. **Built-in type resolution**: The parser converts `Nat` → `(surf-nat-type)` → `(expr-Nat)` (canonical AST). This means session types elaborate to canonical forms, not `(expr-fvar 'Nat)`, which is critical for `check` patterns in `typing-core.rkt` to match.

3. **Nat vs Int literal**: `42` in sexp mode is `(surf-int-lit 42)` → `(expr-int 42)` (type: Int). `42N` is `(surf-nat-lit 42)` → `(expr-nat-val 42)` (type: Nat). Session types must match their literal types.

4. **Unannotated proc-recv**: Process `(proc-recv self x ...)` doesn't annotate x's type — the type should come from the session context. Using `#f` as annotation and adding `(or (not a-annot) ...)` guard in typing-sessions is the correct fix. The `is-type` check for the session's receive type still works because `(expr-Nat)` is a canonical type that `infer` returns `(expr-Type (lzero))` for, even with empty gamma.

5. **S3c `throws` deferred**: The `throws` desugaring needs capability integration (S5), so it's deferred until capabilities are built.
