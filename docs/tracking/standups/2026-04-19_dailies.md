- **No parity axis maps to Phase 1** per §9.1 (infrastructure phase; no divergence class). Commit message must note "no parity tests enabled; structural enforcement verified via unit tests only."

**Principles in play** (ordered by load-bearing weight for this phase):
1. **Correct-by-Construction** — the whole point of A8. Replace discipline-maintained `:component-paths` rule with structural enforcement at registration.
2. **Decomplection** — separate three concerns (lattice type / merge-fn implementation / cell instance) that were conflated under D.1's cell-level `:lattice` annotation.
3. **First-Class by Default** — merge functions become named registered entities; inline lambdas are pressured to graduate to named merge functions.
4. **Most General Interface** — one `#:domain` override keyword serves all exceptional cases; no anonymous classification tags.
5. **Completeness** — registration covers all 37 production merge functions, not "most of them"; the hard-error flip step makes this structural.

**Mantra check** — running each word:
- *All-at-once*: registration happens at module load (all modules load sequentially anyway); cells inherit automatically at creation. ✓
- *All in parallel*: enforcement is a local per-propagator-installation check, not a scan across cells. ✓
- *Structurally emergent*: the classification flows Tier 1 → Tier 2 → Tier 3 by inheritance; enforcement reads it at propagator install. ✓
- *Information flow*: domain → classification → enforcement decision; all via registered structures. ✓
- *ON-NETWORK* ⚠️: **Tier 2 registry lives as a Racket parameter (not a cell) in 4C** — `register-merge-fn!/lattice` mutates a registry hash. M1 external critique established PM Track 12 owns registries-on-network. Phase 1 MUST label the Tier 2 registry explicitly as scaffolding with a documented retirement plan (PM Track 12). Otherwise this is an unlabeled off-network structure, a Vision Alignment Gate failure waiting to happen.

**Drift risks** (named now, checked at Vision Alignment Gate step 5d):

1. **"Register top-10 and declare done"** — skipping the structural enforcement check at `net-add-propagator`. The point of Phase 1 is the ENFORCEMENT, not just the registration. Without the check, we've built infrastructure that isn't enforced. ⚠️ Watch during implementation.
2. **API shape baked before auditing Tier 1 (SRE domain registration)** — if `register-merge-fn!/lattice` signature is decided before reading `sre-core.rkt`'s `register-domain!` API, Tier 2 may duplicate or conflict with Tier 1 structure. ⚠️ Mini-audit step 2 must cover SRE domain API *before* finalizing Tier 2 API.
3. **Inline lambdas (1%) get `#:domain` escape hatch instead of being named** — the First-Class by Default principle pushes "wrap into named merge function" (preferred per D.3 Phase 1 work step 5); risk is reaching for `#:domain` as the fast fix. ⚠️ Audit the 1 inline lambda specifically; give it a name, don't override.
4. **Unlabeled off-network Tier 2 registry** — see mantra check ⚠️. Registry must be explicitly labeled as scaffolding; label goes in code comment AND Phase 1 commit message.
5. **`tools/lint-cells.rkt` scope creep** — lint tool is supporting infrastructure, not a mini-project. Scope: production-only grep, produce a baseline list, exit non-zero if new unregistered additions appear. No dashboard, no UI, no cleverness.
6. **Structural enforcement as a scan** — if I describe the check as "iterate all cells to find unregistered ones" that's step-think. The check is per-cell at read-time (at `net-add-propagator` call), not scan. ⚠️ Language check during implementation.
7. **Phase completion without tests** — Phase 1 adds registration API + enforcement check. Tests needed: registration works; cell inherits; `#:domain` override works; enforcement fires on structural + missing `:component-paths`; lint tool produces expected baseline. ⚠️ Don't ship code without these tests.
8. **Hard-error flip deferred** — Phase 1 step 7 ("Hard-error flip: once baseline is empty, Tier 2 registration becomes required") is the last step.
