- **NTT Research Doc 1: Categorical Foundations of Typed Propagator Networks** (`68d7302`)
  - **Framework comparison**: Operads (single-output) < PROPs (fixed-arity) < **Polynomial functors** (dependent fan-out, compositional wiring, mode-dependent dynamics). Poly is the foundation.
  - **Propagator networks as polynomial functors**: `p(y) = Σ_{i∈I} y^{B_i}` where positions = output cells, directions = dependency cells. SRE structural decomposition = polynomial dispatch over constructor tags.
  - **Stratification as Grothendieck bifibration**: Cartesian lifts (readiness detection), opcartesian lifts (resolution commitment). Post-SRE, genuine for all five systems.
  - **Kan extensions**: Left Kan = speculative forwarding (bridge propagators), Right Kan = demand-driven (threshold propagators). Adjunction guarantees agreement.
  - **SRE unifies all five systems**: Domain-specific behavior from registered forms (polynomial summands), not different categorical structures. The Five Systems analysis documented a mess of different structures; the SRE makes them uniform.
  - **5 open questions**: dependent polynomials in slice categories, non-monotone strata as reindexing functors, cross-domain quantale morphism verification, e-graph as position-equivalence quotient, dual (lattice + tropical) enrichment.
  - **Key insight**: PUnify's `get-or-create-sub-cells` IS the polynomial functor dispatch. Each constructor → summand: Pi = y², Sigma = y², App = y², PVec = y¹, Map = y², Set = y¹. Adding a form to the SRE registry = adding a polynomial summand.
  - **Lesson**: The categorical formalization didn't create new structures — it named the structures that already exist in our codebase. This is the right approach: identify what's there, give it precise names, then use those names to catch composition errors as type errors.
