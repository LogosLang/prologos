**Lessons from Phases 1-3**:
- The dual-path pattern (BSP → emit, DFS → inline) is clean: ~10-15 lines per fire function, zero changes to existing functions (`sre-decompose-generic`, `install-narrowing-propagators`, `eval-rhs`). The topology stratum calls the SAME functions the fire functions used to call — just from outside the BSP round.
- Cell-id 0 convention avoids struct changes to `prop-network`, which would trigger the pipeline exhaustiveness checklist (14 files). Convention over configuration.
- The D.4 external critique finding (cell-id collision) justified the extra complexity of emitting rule requests. Without it, Phase 3 would have been 5 lines (guard relaxation). With it, Phase 3 is ~25 lines but architecturally correct.

- **PAR Track 1 Phase 4** (`775de00`→`f735c5d`) — Topology stratum in BSP loop. Three bugs found and fixed via diagnostic protocol:
  1. **Decomp-request cell not captured by BSP diff** — `fire-and-collect-writes` only diffed declared outputs. Fix: also check cell-id 0 (request cell).
  2. **Infinite loop from monotone clearing** — `net-cell-write(cell, empty-set)` with set-union merge is a no-op. Fix: `net-cell-reset` bypasses merge (non-monotone, permitted outside BSP).
  3. **decomp-key format crash** — narrowing pair-keys used ad-hoc lists, incompatible with `decomp-key-hash` (expects cell-id pairs). Fix: use `decomp-key()` constructor.
  - **Narrowing: 53/53 GREEN under BSP.** All three narrowing test files pass.
  - **SRE: 24/26 subtype, 6/17 duality.** Remaining failures need investigation — compound subtype contradiction not propagating.

**Lesson**: The diagnostic protocol (audit → hypothesize → test narrowly) caught three distinct bugs in one session. Each was found by tracing the simplest failing case (`not(true) = false`) step by step. Without the protocol, these would have been whack-a-mole.
