## Considerations

- Prelude loading is lazy (first eval, not server start) — avoids startup delay
- Session state snapshots 11 parameters back after each eval — matches test-support.rkt exactly
- Form detection is text-based (col-0 convention) — no tree-sitter WASM needed
- Decorations clear on any document edit — prevents stale results

## Lessons Learned

- `process-string-ws` already formats results via `pp-expr` — no custom formatting needed
