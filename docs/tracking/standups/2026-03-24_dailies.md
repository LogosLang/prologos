## Design Decisions

- **Stage 2 audit as separate document**: Applied the PM 8F methodology lesson — concrete code measurements in a standalone audit doc, feeding the Stage 3 design. This prevented the vision from contaminating the audit.
- **CHAMP fork as isolation mechanism**: Structural sharing via persistent data structure, not parameterize scoping. Copy-on-write guarantees isolation. O(1) fork. Composes transitively (prelude → file → command → speculation).
- **Pre-0 benchmarks BEFORE design iteration**: Applied the "benchmark as design input" methodology. The D.1 draft was written with targets; the benchmarks will validate or invalidate those targets before D.2.

## Lessons

- **"Pre-existing issue" is a process smell**: The REPL hang was dismissed for multiple sessions. Investigation cost: 10 lines, 30 minutes. Workaround cost: 2+ hours. Codified as workflow rule: file immediately, estimate workaround cost, escalate if >30min/session.
- **Prelude loading tax is structural, not algorithmic**: Adding match arms to pipeline files slows ALL module loads. The fix is architectural (persistent prelude), not tactical (reorder arms). Every new AST node compounds the debt.
- **The elaborator was already store-agnostic**: SRE Track 2 estimated 158 typing-core changes; actual was 0. The audit-first methodology caught this, but the VISION (elaborator-as-thin-walker) initially overwrote the AUDIT (elaborator-already-delegates).
