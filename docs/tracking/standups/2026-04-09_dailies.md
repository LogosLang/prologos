### D.10: Phase 5 compound cell architecture design (commit `d66e55f7`)

Design conversation → 7 design decisions incorporated into D.10:

1. **Compound decisions cell**: All decision state in ONE cell, component-indexed by group-id. Merge maintains derived bitmask (OR of committed assumptions' bits). The merge IS the fan-in — no micro-propagators, no centralized aggregator. Retraction handled naturally (merge recomputes from all components, removed bits disappear). M separate cells → 1 compound cell.

2. **Compound commitments cell**: Same pattern for per-nogood commitment tracking. K separate cells → 1 compound cell, component-indexed by nogood-id.

3. **Broadcast #:component-paths extension**: `net-add-broadcast-propagator` extended with same interface as `net-add-propagator` (#:component-paths, #:assumption, #:decision-cell). Required for commit-tracker watching compound decisions cell components.

4. **Fire-once audit**: Nogood narrower + contradiction detector identified as fire-once candidates (they produce output once, then are no-ops). `net-add-fire-once-propagator` moved from typing-propagators.rkt to propagator.rkt as general infrastructure.

5. **Worldview cache wiring**: ONE projection propagator (compound decisions → cell-id 1). O(1) per fire. Cache merge changed from bitwise-ior to replacement (projection writes complete recomputed bitmask).

6. **Consumer migration (from deferred 4c)**: elab-speculation-bridge writes to compound decisions cell → projection updates cache → auto-tagged writes → emergent commit/retract. cell-ops and metavar-store read cache cell. typing-propagators deferred to Phase 6.

7. **§2.6b reconciliation**: "No worldview cell" → "Worldview emerges via compound cell." Architecture summary updated.
