### Efficient Nat: Native Integer Representation (Idris 2 Model)

Replaced pure Peano encoding (O(n) suc-chains) with native integer representation
(`expr-nat-val`) while preserving Peano surface syntax. Three-phase implementation:

**Phase A: Purely additive AST pipeline** (commit `f17c522`)
- New struct `(struct expr-nat-val (n) #:transparent)` in syntax.rkt
- Added to all 14 pipeline files: substitution, zonk, typing-core, qtt, pretty-print,
  unify (8 cross-repr cases), foreign, reduction, type-lattice
- All existing tests pass unchanged

**Phase B: Core behavior switch** (commit `c2ad2b5`)
- `nat->expr` and `racket-nat->expr` → `(expr-nat-val n)` (O(1) vs O(n))
- `nat-value` O(1) fast path (benefits all 25 call sites in reduction.rkt)
- Elaborator: `surf-zero` → `(expr-nat-val 0)`, `surf-suc` collapses concrete predecessors
- whnf: natrec iota for `expr-nat-val`, suc collapse `(expr-suc (expr-nat-val k))` → `(expr-nat-val (k+1))`
- nf: `(expr-zero)` → `(expr-nat-val 0)`, suc-chains collapse to `(expr-nat-val n)`
- Pattern matching: `expr-nat-val` zero/suc in `try-builtin-reduce`

**Phase C: Test fixes** (commit `5ef9fed`)
- 17 test files updated (49 assertions), all mechanical expected-value changes
- Full suite: 5005 tests, 246 files, all pass (157.7s)

**Performance impact**: O(1) Nat construction, storage, and extraction. `(expr-nat-val 8080)`
= 1 struct instead of 8081 nested structs. Critical for theorem proving, property-based
testing, and `??` editor-assisted interactions.
