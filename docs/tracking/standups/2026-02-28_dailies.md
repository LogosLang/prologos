## Literate Book System — Phase 2: Weaver Prototype — Complete

Built `racket/prologos/tools/weave-stdlib.rkt` (~425 code LOC) — the dual of the
tangler. While the tangler extracts compilation units for the machine, the weaver
renders the full literate chapter (prose + code together) as human-readable HTML.

### What it does

- Reads OUTLINE manifest (with part-grouping comments)
- Parses chapter `.prologos` files line-by-line via state machine
- Recognizes 6 structural patterns: heavy fences (chapter title), double fences
  (part headers), section headers (`──`), module directives, prose (`;; `), code
- Renders HTML with:
  - Chapter title as `<h1>`, part headers as `<h2>`, section headers as `<h3>`
  - Prose paragraphs with inline formatting (`**bold**` → `<strong>`, `` `code` `` → `<code>`)
  - Code blocks with proper HTML escaping (`<Bool>` → `&lt;Bool&gt;`)
  - Module badges (small pills showing which compilation unit code belongs to)
  - Prev/TOC/Next navigation at top and bottom
  - Book-like CSS: Georgia serif, 740px max-width, left-border code blocks
- Generates `docs/stdlib-book/index.html` (TOC with part groupings) + per-chapter pages

### Architecture

- 6 element structs for parsed chapter IR + 2 TOC structs
- Single-pass state machine parser (states: preamble, fence-title, fence-part, in-module)
- Flush-at-boundary pattern: accumulate prose/code, flush to IR elements on transitions
- Inline formatting via `html-escape` then `regexp-replace*` for bold/code markers
- CSS embedded in each page (self-contained, works offline)
- CLI: `racket tools/weave-stdlib.rkt [--book-dir DIR] [--output-dir DIR]`

### Fixes during implementation

- **Regex engine**: `#rx` (basic regex) couldn't handle `[─]` character class with Unicode
  box-drawing chars. Switched section header detection to `#px` (Perl-compatible regex).
- **Title leading space**: Chapter title extraction had a leading space from regex capture.
  Fixed with `string-trim` on the extracted title.

- All structural elements render correctly (10 section headers, 7 module badges, 2 part headers)
- HTML escaping works (angle brackets in type annotations, arrows in match expressions)
- Inline prose formatting works (bold, backtick code, Unicode math symbols)

## Literate Book System — Phase 5: Spec Pre-Scan — Complete (commit 49be0b8)

Added a pre-scan loop in `preparse-expand-all` (macros.rkt) that registers all
`spec` annotations before the main expansion pass. This enables defn-before-spec
ordering within a module — a defn can appear before its matching spec, and
`maybe-inject-spec` will still find the type annotation.

### Implementation

- **macros.rkt**: ~20 LOC pre-scan loop before the main `for/fold`. Iterates all
  forms, recognizes `spec` and `spec-` (private), calls `process-spec` +
  `maybe-register-trait-dict-def` + `auto-export-name!`. All three operations are
  idempotent, so the main loop's re-processing is harmless.
- **test-spec-ordering.rkt** (NEW): 10 test cases covering baseline (spec-before-defn),
  forward references (defn-before-spec), two-param, Bool result, multiple pairs,
  mixed ordering, docstrings, chained defns, higher-order types.
- **dep-graph.rkt**: Added test entry with same deps as test-spec.rkt.

### What this does NOT address

- Data forward references (defn referencing constructor from later `data`)
- Trait forward references (similar issue with `process-trait`)
- Cross-module forward references (handled by `require` ordering)

## Test Runner Fix (commit 412328b)

Fixed a longstanding bug in `tools/bench-lib.rkt` where the test runner silently
failed when `raco` was not on PATH. `find-executable-path "raco"` returned `#f`,
and `subprocess` with `#f` as the executable silently exits 1 — causing every
test to report "FAIL" with 0 tests in 0.0s.

**Root cause:** `raco` lives at `/Applications/Racket v9.0/bin/raco` but this
directory isn't on PATH. The runner was spawning subprocesses that immediately
failed.

**Fix:** Added `find-raco-path` helper that falls back to deriving the raco
location from `(find-system-path 'exec-file)` — the currently running racket
binary's sibling. Cached as a module-level binding (computed once at load time).

**Verification:** `run-affected-tests.rkt --all` now works correctly:
4632 tests in 401s, all pass. Whale files (>30s): test-reduction-perf-02-01 (80.7s),
test-trait-impl-04 (57.8s), test-transducer (57.3s) — pre-existing, not new.

## Shared Fixture Conversion (commit 0727fd6)

Converted all 14 whale test files from per-test `run-ns`/`run-ns-last` (fresh env
per call) to the shared fixture pattern: modules loaded ONCE at module level via
`define-values`, each test reuses cached `shared-global-env`/`shared-ns-context`/
`shared-module-reg` via `parameterize` with fresh `current-mult-meta-store`.

### Pattern

Uses `prelude-module-registry` from `test-support.rkt` as starting registry for
fast cached module lookups (not `(hasheq)` which would reload all ~84 prelude
modules from source). Each file's shared preamble processes `(ns test)` + specific
`(require ...)` once; individual tests only process their eval/check/infer expression.

### Conversion approach

1. Extracted union of all per-test `(require ...)` into a single `shared-preamble`
2. Built `define-values` fixture starting from cached `prelude-*` registries
3. Replaced `run-ns-last (string-append "(ns ...)\n(require ...)\n" test-code)` with
   `run-last test-code`
4. All test assertions preserved exactly (no content changes)
