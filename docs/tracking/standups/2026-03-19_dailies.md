#### `defr |` Literal Pattern Fix (commit `490a4e3`)
- **Root cause**: `parse-rel-params` (parser.rkt) only accepted symbols; literal values (ints, strings, bools) in `defr |` variant params caused a parse error that silently prevented relation registration.
- **Fix**: `parse-rel-params` now returns `(#:literal . value)` for non-symbol elements. `elaborate-defr-variant` desugars these to fresh gensym'd logic vars + implicit `(= var literal)` unification goals prepended to each clause body.
- **Key insight**: `defr |` with all-variable params always worked — only literal patterns were broken.

#### PUnify Part 2: Lattice-First Architecture Refinement (commit `349c9d2`)
- Extended design doc with lattice-first architecture (§5.5) and grounded cost analysis (§5.6)
- **Type constructor descriptor registry**: `type-ctor-desc` struct with tag, arity, extract-fn, reconstruct-fn, component-lattices, binder-depth. Single registration site replaces three independent structural knowledge encodings (try-unify-pure ~170 lines, classify-whnf-problem ~150 lines, maybe-decompose ~100 lines).
- **Recursive sum-of-products lattice**: TypeLattice = Lifted(FlatLattice(Atom) ⊔ Σ_tag ∏ L_i). Compositional monotonicity: product lattice correctness follows from component correctness.
- **Grounded cost**: ~8-11ms overhead (0.05% of 17.9s baseline). 2,500 cells, 8,500 CHAMP ops, 4,000 struct-copies. Well within 10% budget.
- **Design rationale**: Lattice framing aligns with propagator-first principles, provides compositional correctness guarantees, creates vocabulary for Track 8 second half (mult/level as component lattices), and reduces pipeline exhaustiveness surface. This is a design discipline over the same runtime operations as ad-hoc Option C.
- Phases 3-5 collapsed into 3-4: once Pi works via descriptors, remaining compound types are registration + testing, not new structural code.
- Key theoretical connections grounded in existing code: `term-lattice.rkt` already implements the tree-of-cells pattern (term-ctor with sub-cells); PUnify extends this to type-level unification. The flat/tree Galois connection (α=reconstruct, γ=decompose) formalizes the on-demand pattern.
