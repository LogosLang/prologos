## Design Principle

Nat is for type-level infrastructure (indices, lengths, inductive proofs), NOT general numerical computation. Int/Rat/Posit are the computational numeric types.

## Performance Improvements

Previously failing computations now work:
- `fib 20` = 6765 (previously hung at fib 15+)
- `fib 30` = 832040
- `fact 20` = 2432902008176640000 (previously capped at 10)
- `collatz-len 27` = 111 steps (previously timed out)
- `e-approx 7` = 685/252 (nested recursion previously hung)
- `golden-approx 15` = 987/610

## Key Changes

### reduction.rkt
- **nf-whnf**: Lazy branch normalization for boolrec, natrec, J — only normalizes scrutinee; if resolved, takes appropriate branch; otherwise leaves branches unnormalized
- **WHNF cache**: `current-whnf-cache` parameter, same pattern as existing NF cache
- **Fuel**: `current-reduction-fuel` parameter (box of step count), 1M default in driver
- **nat-value cache**: `current-nat-value-cache` for O(1) repeated Nat→Int coercion
- **Posit dedup**: 8 functions → 2 (`reduce-posit-binary`, `reduce-posit-unary`) parameterized by width

### reader.rkt
- `read-number-token!` extended for decimal point: `3.14` → exact rational `157/50`
- `~3.14` automatically works via existing tilde handler

### driver.rkt
- `process-command` activates all caches and fuel: `current-nf-cache`, `current-whnf-cache`, `current-nat-value-cache`, `current-reduction-fuel`
