# Phase 5: Persistent ATMS — Hypothetical Reasoning

**Date**: 2026-02-25
**Status**: COMPLETE

## Overview

Persistent Assumption-Based Truth Maintenance System (ATMS) following de Kleer 1986. Enables hypothetical reasoning: choice points (`amb`), nogood recording, worldview switching, and dependency-directed backtracking. All operations are pure — the old ATMS is never modified (structural sharing via `hasheq`).

## Design Decisions

- **IDs are Nat-wrappers** — `assumption-id` wraps a Nat, like `cell-id` and `prop-id`
- **Backed by `hasheq`** — consistent with Phase 4 (UnionFind), not CHAMP (deferred optimization)
- **Assumption sets as `hasheq`** — `assumption-id -> #t` maps used as sets
- **`amb-groups` tracked explicitly** — the `atms` struct stores a list of amb groups for solve-all enumeration
- **`atms-read` and `atms-solve-all` return `expr-hole`** — type-unsafe (heterogeneous TMS cell values)
- **No `expr-supported-type`** — deferred. Parameterized type for TMS cells left for later
- **No two-tier lazy activation** — Phase 0 always creates full ATMS
- **Sequential solve-all** — BSP parallel worldview exploration deferred
